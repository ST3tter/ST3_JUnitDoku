\chapter{Einleitung}\label{ch:einleitung}
	In vielen Bereichen der Softwareentwicklung ist es heutzutage unumgänglich sogenannte Unit-Tests zu entwickeln, um die Codequalität zu gewährleisten. Mit Hilfe von Unit-Tests kann der Code jederzeit automatisch getestet werden, so dass Fehler bei "Anderungen am Code rechtzeitig und somit kostengünstig behoben werden können.\par
	%
	Im Internet ist eine Vielzahl an Testumgebungen zu finden, die das erstellen von Unit-Tests erleichtern. Eine beliebte und viel genutzte Test-Framework-Familie ist \textsf{xUnit}. In diesem Dokument wird vorwiegend auf das Framework \textsf{JUnit}, welches in der Java-Programmierung eingesetzt wird, genauer eingegangen. Das Test-Framework \textsf{JUnit} bietet dem Benutzer allerhand Funktionen, um möglichst effizient Unit-Tests umzusetzen.\par
	%
	Da Unit-Testing leider kein Platz im Lehrplan der \gls{ac:bfh} bekommen hat, soll diese Einleitung den Studenten einen möglichst einfachen Einstieg in diese Thematik bieten. Nach der Einführung in das Unit-Testing, werden in den darauffolgenden Kapiteln die wichtigsten Schritte und M"oglichkeiten von \textsf{JUnit} kurz erklärt und mit einem Anwendungsbeispiel veranschaulicht. Anschliessend wird auf die Erweiterung \textsf{EclEmma} eingegangen, welche eine gute Erg"anzung zum JUnit-Framework darstellt.
